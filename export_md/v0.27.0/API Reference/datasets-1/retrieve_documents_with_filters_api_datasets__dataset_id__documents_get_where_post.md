excerpt: "Retrieve documents with filters.\nCursor is provided to retrieve even more documents. Loop through it to retrieve all documents in the database.\nFilter is used to retrieve documents that match the conditions set in a filter query. This is used in advance search to filter the documents that are searched.\n\nThe filters query is a json body that follows the schema of:\n\n    [\n        {'field' : <field to filter>, 'filter_type' : <type of filter>, \"condition\":\"==\", \"condition_value\":\"america\"},\n        {'field' : <field to filter>, 'filter_type' : <type of filter>, \"condition\":\">=\", \"condition_value\":90},\n    ]\n\nThese are the available filter_type types: **[\"contains\", \"category\", \"categories\", \"exists\", \"date\", \"numeric\", \"ids\"]**\n\n1. **\"contains\"**: for filtering documents that contains a string.\n        {'field' : 'item_brand', 'filter_type' : 'contains', \"condition\":\"==\", \"condition_value\": \"samsu\"}\n2. **\"exact_match\"/\"category\"**: for filtering documents that matches a string or list of strings exactly.\n        {'field' : 'item_brand', 'filter_type' : 'category', \"condition\":\"==\", \"condition_value\": \"sumsung\"}\n3. **\"categories\"**: for filtering documents that contains any of a category from a list of categories.\n        {'field' : 'item_category_tags', 'filter_type' : 'categories', \"condition\":\"==\", \"condition_value\": [\"tv\", \"smart\", \"bluetooth_compatible\"]}\n4. **\"exists\"**: for filtering documents that contains a field.\n        {'field' : 'purchased', 'filter_type' : 'exists', \"condition\":\"==\", \"condition_value\":\" \"}\nIf you are looking to filter for documents where a field doesn't exist, run this:\n        {'field' : 'purchased', 'filter_type' : 'exists', \"condition\":\"!=\", \"condition_value\":\" \"}\n5. **\"date\"**: for filtering date by date range.\n        {'field' : 'insert_date_', 'filter_type' : 'date', \"condition\":\">=\", \"condition_value\":\"2020-01-01\"}\n6. **\"numeric\"**: for filtering by numeric range.\n        {'field' : 'price', 'filter_type' : 'numeric', \"condition\":\">=\", \"condition_value\":90}\n7. **\"ids\"**: for filtering by document ids.\n        {'field' : 'ids', 'filter_type' : 'ids', \"condition\":\"==\", \"condition_value\":[\"1\", \"10\"]}\n\nThese are the available conditions:\n\n    \"==\", \"!=\", \">=\", \">\", \"<\", \"<=\"\n\nIf you are looking to combine your filters with multiple ORs, simply add the following inside the query\n`{\"strict\":\"must_or\"}`. \n\nFor pagination support refer to **/datasets/{dataset_id}/documents/paginate**."
